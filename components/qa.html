常见问题解答 / Q&A

💡 提示

这里的回答涵盖了博客功能、技术栈以及访问时可能遇到的技术问题。

1. 为什么页面显示 "Failed to fetch" 或加载失败？

这通常是因为触发了 GitHub API 的访问频率限制。本博客通过浏览器前端直接调用 GitHub 接口：

  • 未登录状态： 同一 IP 每小时仅支持 60 次请求，若在短时间内频繁刷新，会导致服务器返回 403 Forbidden。
  • 🚀 解决方法： 点击右上方导航栏的 LOGIN 登录你的 GitHub 账号。登录后，限制将提升至每小时 5000 次，浏览体验更顺畅。

2. 我可以发布文章吗？

不可以。 为了保证内容的质量与纯粹，目前仅博主拥有发布权限。但你可以参与评论：在文章详情页点击“在 GitHub 评论”，即可在相应的 Issue 页面留下你的足迹。

3. 关于“纠错”与反馈按钮

如果你在文章中发现了拼写错误或代码 Bug，欢迎点击文章右上角的 “文章有问题” 按钮。在登录状态下，你可以直接提交反馈，这些信息会同步到 GitHub Issue 中提醒我修改。

4. 音乐播放器没有声音？

由于现代浏览器的自动播放策略，背景音乐无法在进入页面时静默播放：

  • 手动激活： 点击右下角的旋转唱片图标。
  • 切歌操作： 点击音乐标题区域可弹出播放列表。

5. 这是一个开源项目吗？

是的。该博客的全部代码均托管在 GitHub。你可以通过 ABOUT 页面下方的链接跳转到源代码仓库进行查看。欢迎 Star 或反馈建议。

还有其他疑问？欢迎通过 GitHub 提交反馈。
